## Session 9: Full 128-Character Batch Generation

### What was done
- First run hit Anthropic rate limit (10,000 output tokens/min) immediately — 1.2s delay was far too short.
  - 5 succeeded, 15 fell back to mock, rest all rate-limited.
- Fixed the generator script:
  - Increased delay from 1.2s to 20s between Claude calls (3 NPCs/min fits 10k token/min limit).
  - Added 3-retry logic with 60s/120s/180s exponential backoff on rate limit errors.
  - Added `hasRealData()` check to skip files already containing real Claude data.
  - `callClaude` now returns `'RATE_LIMITED'` sentinel for retry differentiation.
- Second run: 61 new + 31 skipped (already real) + 0 mock fallbacks = 92 total real.
  - Combined with earlier successes: 127/128 real, 1 mock (Zuko — JSON parse failure).
- Deleted Zuko mock and re-ran generator. Zuko regenerated successfully: "Prince Azren of House Infernus".
- **Final result: 160/160 NPCs with real Claude-generated data. 810 tests pass.**

### What worked
- 20-second delay between calls: zero rate limit errors across 61 consecutive Claude calls.
- Skip-existing logic: second run took ~20 minutes instead of 40 by skipping 31 already-done characters.
- Retry with backoff: not triggered in second run (20s delay was sufficient), but provides safety net.
- Generator is idempotent: safe to re-run at any time — only regenerates mock/missing files.

### What was surprising or went wrong
- Anthropic's free tier rate limit is 10,000 output tokens/minute — at ~3000 tokens per NPC, that's only 3 NPCs/min. The initial 1.2s delay was wildly insufficient.
- One JSON parse failure (Zuko) out of 128 calls (~0.8% failure rate). The JSON had a syntax error at position 10909.
- Background terminals in VS Code don't reliably pass `Set-Location` — had to use absolute path to script.

### What to do differently next time
- For APIs with output token rate limits, calculate: `(tokens_per_call × calls_per_minute) ≤ rate_limit`. Then add 50% buffer.
- Always build skip-existing logic into generators from the start — makes them idempotent and resumable.
- For long-running generation scripts, run with absolute paths to avoid CWD issues in background terminals.
