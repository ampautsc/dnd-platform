## Session: Initiative-Based Scene Engine (end-to-end)

### What was done
Built a complete turn-based social scene system from scratch, end-to-end in one pass:

**New files created (DM package — `packages/dm/`):**
- `src/services/SceneState.js` — Immutable state container (modeled on combat's GameState). All `with*()` methods return new frozen instances. Tracks participants, initiative order, turn index, round, transcript, status.
- `src/services/SceneInitiative.js` — Pure function: CHA mod + d20 initiative roller. Sorted descending, ties broken by higher CHA mod. Injectable dice for testing.
- `src/services/SceneEngine.js` — The core turn orchestrator. Creates scenes, rolls initiative, handles `submitAction` (records player → auto-resolves all NPC turns → returns batch). NPC turns call the LLM via `CharacterResponseService` with scene-aware prompts. `parseNpcAction()` extracts `[SPEAK]`/`[ACT]`/`[OBSERVE]`/`[PASS]` from LLM output.
- `src/npc/buildSceneSystemPrompt.js` — Extends the vessel-surrender encounter prompt with scene-specific sections: participants list (no NPC/player labels), round number, turn instructions.
- `__tests__/services/SceneState.test.js` — 18 tests: immutability, all with*() methods, helpers.
- `__tests__/services/SceneInitiative.test.js` — 6 tests: sorting, CHA-based, ties, injectable dice.
- `__tests__/services/SceneEngine.test.js` — 16 tests: create/start/get/submit/end, NPC auto-advance, round cap, wrong-turn rejection, transcript integrity.
- `__tests__/npc/buildSceneSystemPrompt.test.js` — 9 tests: no NPC/player labels, round number, action options, self-exclusion from participant list.

**New files created (API package — `packages/api/`):**
- `src/services/SceneController.js` — Thin bridge (same pattern as EncounterController). Zero logic, pure delegation + toJSON().
- `src/routes/scenes.js` — REST endpoints: POST create, POST start, GET state, POST action, POST end, GET list.
- `__tests__/routes/scenes.test.js` — 11 integration tests with supertest.

**Modified files:**
- `packages/dm/src/index.js` — Exports SceneEngine, SceneState, rollSceneInitiative, buildSceneSystemPrompt. createDmEngine now creates and exposes `sceneEngine`.
- `packages/api/src/app.js` — Optional mount of `/api/scenes` routes.
- `packages/api/src/index.js` — Creates and wires SceneController from dmEngine.sceneEngine.

**Test results:** 2,542 tests passing across 77 files (60 new tests, 0 regressions).

### Key design decisions
- **CHA-based initiative** — social dominance = who speaks first.
- **Strict turn order** — no reactions/interjections. Simpler, predictable LLM call count.
- **NPCs can pass/observe** — LLM decides; response parsed via `[TYPE]` prefix.
- **Auto-advance NPC turns** — when player submits, all NPC turns resolve before returning. Client gets a batch.
- **No NPC/player distinction in prompts** — `isPlayer` is engine bookkeeping only. System prompts list "others present" with names only.
- **Immutable SceneState** — mirrors combat's GameState pattern. Object.freeze in constructor. All mutations return new instances.
- **Coexists with encounters** — existing `/api/encounters` 1:1 chat stays. New `/api/scenes` is parallel.
- **Package boundary clean** — SceneEngine in `dm`, thin bridge in `api`. No `combat` imports.

### What worked
- TDD sequence: tests first, then implementation. Every component passed on first or second run.
- Reusing existing infrastructure (CharacterResponseService, EncounterMemoryService, buildEncounterSystemPrompt) made the engine thin.
- The MockProvider + injection pattern made integration tests trivial.

### What was surprising or went wrong
- `Object.freeze(this)` in SceneState meant `state._createdAt = Date.now()` post-construction failed silently. Fixed by adding `createdAt` as a proper constructor parameter.

### What to do differently next time
- Always plan for frozen objects — never add properties post-construction. If it needs to be on the object, it goes in the constructor params.
- When building a new engine, start from the immutable state container tests — they flush out freezing issues early.
