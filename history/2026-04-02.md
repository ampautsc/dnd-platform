## Session 1: Cross-NPC Cache Sharing — Sonnet 4.6 Confirms Prefix Sharing

### What was done
Ran `test-cross-npc-cache.mjs` with Sonnet 4.6 (`claude-sonnet-4-6`, 1,024 token minimum) instead of Haiku 4.5 (4,096 minimum). World knowledge block is ~1,903 tokens — above Sonnet's threshold but below Haiku's.

### Results
| Call | NPC | cache_creation | cache_read | Total system tokens |
|---|---|---|---|---|
| 1 | Samren | 7,833 | 0 | 7,833 |
| 2 | Mira | 3,027 | 2,162 | 5,189 |
| 3 | Mira | 0 | 5,189 | 5,189 |

### Analysis
**Cross-NPC prefix sharing works.** Call 2 proves it:
- `cache_read: 2,162` — world knowledge prefix was found in cache from Call 1
- `cache_creation: 3,027` — Mira consciousness was written as new cache entry
- The lookback mechanism found the block 1 breakpoint entry and reused it

This confirms the Anthropic docs: prefix sharing across requests with different suffixes IS supported, provided each breakpoint independently exceeds the model's minimum threshold. Haiku 4.5 failed because 1,903 < 4,096. Sonnet 4.6 succeeded because 1,903 > 1,024.

### Cost implications
With Sonnet 4.6 and cross-NPC sharing:
- First NPC call: pays full cache write (1.25x) for both blocks
- Each new NPC: reads block 1 (0.1x base) + writes block 2 (1.25x)
- Repeat same NPC: reads both blocks (0.1x base)
- Savings grow with more NPCs sharing the same world knowledge prefix

### What to do differently
- For Haiku 4.5: world knowledge must be 4,096+ tokens to enable cross-NPC sharing
- For Sonnet 4.6/4: 1,024+ tokens suffices — current world knowledge works
- Model choice for NPC calls should factor in caching threshold, not just per-token price
- Sonnet 4.5 model ID (`claude-sonnet-4-5-20250514`) returned 404 — it's legacy. Current Sonnet is 4.6 (`claude-sonnet-4-6`)

## Session 3: worldKnowledge.xml Rewrite — Commoner Perspective, Canon Only

### What was done
Completely rewrote `worldKnowledge.xml` and its test file based on a long refinement process. The old version (Session 2) included many things a commoner wouldn't know (8 schools of magic, the giant ordning, alignment codes, mind flayers, detailed outer plane cosmology, the Fugue Plane, Wall of the Faithless). It also included "trade language" for Common, which is NOT canon — PHB pg. 123 lists Common as the typical language of Humans.

The user drove 10+ outline iterations before approving a final 7-category structure.

### Approved outline (7 categories)
1. Currency & Cost of Living (`<currency>`)
2. The Social Order (`<social_order>`)
3. The Physical World (`<physical_world>`)
4. The Gods & The Afterlife (`<gods_and_afterlife>`)
5. Magic (`<magic>`)
6. The Peoples (`<peoples>`)
7. Monsters & The Wilderness (`<monsters_and_wilderness>`)

### Key design principle
"Common knowledge is different than knowledge." Every fact is filtered through what a COMMONER would actually know — not what a scholar, wizard, or adventurer would know.

### Facts researched and verified against official sources
- **PHB Ch.5 / Basic Rules Equipment**: Coinage exchange rates, lifestyle expenses, food/drink prices, trade goods, mount prices, vehicle prices, services costs, hiring costs
- **PHB Ch.2 / Basic Rules Races**: All 9 PHB race descriptions, lifespans, heights, traits, subraces
- **PHB Ch.4 / Basic Rules Languages**: Language tables, Common as language of Humans
- **Basic Rules Appendix B / SCAG**: Forgotten Realms deity list with domains

### What was deliberately EXCLUDED (commoner filter)
- Schools of magic (abjuration, evocation etc.) — commoner wouldn't know them
- Alignment codes on gods (LG, LE, etc.) — game mechanic, not commoner knowledge. Grouped as good/neutral/evil instead.
- Monster taxonomies — no beholders, mind flayers, Far Realm. Commoners fear goblins and trolls, not aberrations.
- Detailed outer planes — no Mount Celestia, Mechanus, Limbo, Elysium, Arborea, Hades. Commoner knows "heavens, hells, and stranger places."
- Giant ordning — commoner knows hill giants are dumb and dangerous, not the formal caste system by name.
- Fugue Plane / Wall of the Faithless — too specific. Replaced with "soul passes on, faithless face a grim fate."
- "Trade language" for Common — this is NOT canon. PHB lists Common as the typical language of Humans. "Trade language" is community headcanon.
- Calendar (tendays) — Forgotten Realms specific, not universal 5e

### Files changed
- **`packages/dm/src/prompts/worldKnowledge.xml`** — Complete rewrite from 13 sections to 7 sections. Every fact verified against PHB/Basic Rules/SRD.
- **`packages/dm/__tests__/prompts/worldKnowledge.test.js`** — Complete rewrite: 73 tests. Tests verify: structure (7 required sections, no old sections), currency facts (exchange rates, purchasing examples, barter, food prices, trade goods, guilds), social order (hierarchy, law enforcement, justice, literacy, Common NOT "trade language", racial languages), physical world (travel pace, mount prices, Material Plane, Feywild/Shadowfell, NO detailed cosmology), gods (real gods, pragmatic polytheism, major deities by domain, patron deities, NO alignment codes, resurrection costs), magic (rare, Weave/Mystra, caster types, NO schools, service costs, potion prices), peoples (all 9 PHB races, lifespans, elf trance, subraces, halfling traits, dragonborn breath, tiefling fire/mistrust), monsters (common threats, undead, dragon types, devils vs demons, NO exotic monsters, troll fire weakness), well-formed XML, commoner perspective filter (no alignment codes, no schools, no "trade language", no outer planes, no Fugue Plane, no Wall, no ordning).

### Test results
73/73 worldKnowledge + 15/15 millhavenKnowledge = 88/88 all passing.

### What worked
- Researching official sources FIRST (D&D Beyond Basic Rules fetch) then writing — zero fabricated facts.
- The commoner perspective filter removes a LOT of content that seems essential but really isn't. A commoner doesn't need to know 8 schools of magic to be a believable NPC.
- 10 outline iterations with the user, while painful, produced a much tighter scope than my first attempt.

### What was surprising
- "Common is a trade language" is deeply entrenched as fan lore but is NOT in the PHB. The PHB simply lists Common as the typical human language. This was a hard lesson in "verify everything."
- The user's standard for "commoner knowledge" is much narrower than I initially assumed. I kept including scholar/adventurer knowledge (schools of magic, planar cosmology, monster ecology) and had to learn that commoners genuinely wouldn't know these things.

### What to do differently
- Start with "what would a farmer in a village actually know?" as the filter question, not "what are the key facts of D&D 5e?"
- Never include a fact without being able to cite the specific PHB/SRD page. If it's "common fan knowledge" but not in an official source, it's trash.
- When the user rejects work, go all the way back to outline FIRST before rewriting content. Writing content against a bad outline is wasted effort.
