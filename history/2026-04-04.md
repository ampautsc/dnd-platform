### Why this was needed
User wants to host the platform on Azure. The provision workflow automates the full Azure setup тАФ no manual portal clicking needed.

### Why merge was required

### What worked
- `azure/login@v2` + `az` CLI for all resource creation
- `gh` CLI (via GH_PAT secret) to write GitHub secrets/variables automatically from within the workflow
- Graceful degradation: if a resource already exists, the workflow continues rather than failing

### What worked
- `mock.timers.enable({ apis: ['setInterval'] })` тАФ only list the creation function, NOT `clearInterval`. `clearInterval` is not an accepted api name in Node.js mock.timers.
- Pure reducer pattern for villain storylines and location state тАФ tests are trivial (no engine, no timers, just fn(input) тЖТ output)
- Keeping location weather/season as computed-on-query rather than stored reduces state management complexity

### What was surprising
- Node.js v24 `mock.timers` does not accept `clearInterval` in the `apis` array тАФ only `setInterval`, `setTimeout`, `setImmediate`, `Date` are valid. Passing `clearInterval` throws `ERR_INVALID_ARG_VALUE`.
