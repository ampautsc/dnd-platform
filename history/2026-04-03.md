# 2026-04-03

## Session 1: worldKnowledge.xml — Subsection Restructuring for LLM Retrieval

### What was done
Restructured worldKnowledge.xml from flat sections (7 walls of prose) into semantically organized subsections (3 levels deep per xml-prompt-engineering.md rules). Same content, better structure for model retrieval.

### Subsection structure added
| Parent | Subsections |
|---|---|
| `<currency>` | `<coins>`, `<cost_of_living>`, `<commerce>` |
| `<social_order>` | `<hierarchy>`, `<governance>`, `<languages>` |
| `<physical_world>` | `<travel>`, `<planes>` |
| `<gods_and_afterlife>` | `<worship>`, `<pantheon>`, `<death>` |
| `<magic>` | *(unchanged — stays flat, only ~130 words, single cohesive topic)* |
| `<peoples>` | `<humans>`, `<dwarves>`, `<elves>`, `<halflings>`, `<gnomes>`, `<half_elves>`, `<half_orcs>`, `<dragonborn>`, `<tieflings>` |
| `<monsters_and_wilderness>` | `<common_threats>`, `<undead>`, `<dragons>`, `<greater_threats>` |

### Why this helps
Each subsection tag acts as a semantic anchor for the model. When an NPC meets an elf, the model homes in on `<elves>` instead of scanning all 220 words of `<peoples>`. Tag cost is ~2 tokens per open/close pair — negligible vs. retrieval precision gain. Still within the 3-level-deep maximum (world_knowledge → section → subsection → prose).

### Files changed
