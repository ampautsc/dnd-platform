# Skill: Continuous Learning Protocol

## Category
learning

## Tags
#learning #reflection #protocol

## Description
How to systematically capture and reuse knowledge across work sessions. Every completed task is a potential skill. Every surprise is a lesson. Every failure is a pattern to avoid.

## Steps

### Before Starting Work
1. Search the Skills & Knowledge Index for tags matching the current task
2. If a skill exists, load it and follow it
3. If no skill exists, note that this is new territory

### During Work
1. Note what surprises you
2. Note what doesn't work the way you expected
3. Note workarounds you discover
4. Note things you'd tell someone doing this for the first time

### After Completing Work
1. Update history file with what was done, what worked, what didn't
2. If you followed clear, repeatable steps → create a skill file
3. Add the skill to the index with tags
4. Verify the index is current

### Skill Creation Threshold
- If it has clear steps → it's a skill NOW
- Don't wait for 3 occurrences
- An imperfect skill is infinitely better than no skill

## Common Pitfalls
- "I'll document this later" → you won't
- Creating a skill but not adding it to the index → invisible
- Writing skills that are too vague to follow → useless
- Not recording failures → doomed to repeat
